Description

This change provisions two Amazon S3 buckets to support Netskope Transaction Log Streaming. Netskope is deprecating its existing transaction event streaming service and requires customers to stream transaction logs to customer-managed storage targets. These S3 buckets will serve as the destination for Netskope transaction logs to ensure continuity of logging and compliance with vendor requirements.

Justification ‚Äì Why is the change needed?

Netskope is phasing out its current transaction event streaming service and transitioning customers to stream transaction logs directly to privately managed storage targets. To maintain uninterrupted transaction logging, monitoring, and compliance visibility, AWS S3 buckets must be created and configured before the service deprecation deadline. Failure to implement this change may result in loss of transaction log data and reduced security monitoring capability.
