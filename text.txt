Change Title

Implement AWS S3 and SQS Configuration to Forward Security Logs to Onum

Description

This change implements AWS-side configurations required to enable secure forwarding of AWS security and audit logs (CloudTrail, CloudWatch, and AWS WAF) to the Onum security analytics platform.

The change includes creation and configuration of Amazon SQS queues, IAM roles and policies, and Amazon S3 event notifications. No changes will be made within the Onum platform itself; Onum configuration will be handled separately by the Information Security team using details produced from this implementation.

Justification (Why is the change needed?)

This change is required to enable centralized ingestion of AWS security logs into the Onum platform for enhanced monitoring, threat detection, and compliance visibility.

Forwarding CloudTrail, CloudWatch, and WAF logs allows the Information Security team to analyze AWS activity events, security alerts, and web application firewall logs in near real-time. This improves incident detection capabilities and aligns with organizational security monitoring and audit requirements.
