ðŸ”¹ Change Description

A new Active Directory security group named SNOWFLAKE_SPROUTSOCIAL_ANALYST has been created and synchronized to Microsoft Entra ID.
This change involves assigning the group to the Snowflake Enterprise Application in Entra to enable automatic provisioning through SCIM (System for Cross-domain Identity Management).

The integration ensures that user accounts and access memberships within this group are automatically provisioned and deprovisioned in Snowflake, aligning with organizational identity lifecycle management policies.

The S2SNOWFLAKE_POWERBI service account will also be included in the configuration as requested. Once completed, the task will be reassigned to the Cloud Services team for post-integration validation.

ðŸ”¹ Justification

This change is required to enable automated user provisioning and lifecycle management between Microsoft Entra ID and Snowflake for the newly created SNOWFLAKE_SPROUTSOCIAL_ANALYST security group.

Enabling SCIM integration ensures that group membership changes in Active Directory and Entra ID are automatically reflected in Snowflake, reducing manual administration, eliminating access discrepancies, and improving security compliance.

This integration aligns with corporate identity governance policies and supports least privilege access by ensuring only authorized users can access Sprout Social data sources in Snowflake.
