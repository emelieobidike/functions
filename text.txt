✅ Change Title

Implementation of SAML Single Sign-On (SSO) for qTest via Microsoft Entra ID

⸻

📝 Description

Implementation of SAML-based Single Sign-On (SSO) integration between qTest and Microsoft Entra ID to enable secure, centralized authentication. This will allow users to access qTest using their corporate credentials, improving security, user experience, and identity governance.

⸻

📌 Justification (Why is the change needed?)

qTest currently requires separate authentication or lacks centralized identity management. Implementing SAML SSO with Microsoft Entra ID will:
	•	Improve security through centralized authentication and conditional access policies
	•	Reduce password fatigue and support overhead
	•	Enable seamless user access using corporate credentials
	•	Support compliance with identity and access management standards

⸻

⚙️ Implementation Plan
	1.	Prepare qTest for SSO
	•	Verify qTest supports SAML 2.0
	•	Obtain qTest Service Provider (SP) metadata (ACS URL, Entity ID, certificate)
	2.	Create Enterprise Application in Azure
	•	Navigate to Entra ID → Enterprise Applications
	•	Create new Non-gallery application (qTest SSO)
	3.	Configure SAML in Azure
	•	Set:
	•	Identifier (Entity ID)
	•	Reply URL (ACS URL from qTest)
	•	Upload or configure signing certificate
	4.	Download Federation Metadata XML
	•	Provide metadata to qTest
	5.	Configure qTest IdP Settings
	•	Upload Azure metadata
	•	Map required attributes (e.g. email, name, username)
	6.	Assign Users/Groups
	•	Grant access via Entra ID groups
	7.	Testing
	•	Perform SSO login tests with pilot users
	•	Validate authentication and attribute mapping
	8.	Communication
	•	Notify users of new SSO login method

⸻

⚠️ Risk and Impact Analysis
	•	Impact: Users of qTest platform
	•	Risks:
	•	Misconfiguration may prevent login
	•	Incorrect attribute mapping may cause access issues
	•	Mitigation:
	•	Test with pilot users before rollout
	•	Keep local/admin login as fallback
	•	Downtime: Minimal to none expected

⸻

🔄 Backout Plan
	•	Disable SAML SSO configuration in qTest
	•	Revert users to local authentication
	•	Remove or disable Azure Enterprise Application
	•	Rollback time: ~15–30 minutes

⸻

	•	No disruption to existing services
